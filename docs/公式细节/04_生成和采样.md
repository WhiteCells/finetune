# 生成和采样

## 1. 从 logits 到概率

模型在当前步骤输出词表中每个 token 的 logit `z_i`。softmax 给出概率：

```math
p_i = \frac{e^{z_i}}{\sum_j e^{z_j}}
```

生成过程不断把已生成 token 接回输入：

```math
x_{t+1} \sim p_\theta(\cdot \mid x_{\le t})
```

直到生成 `eos_token_id` 或达到 `max_new_tokens`。

`inference.py` 在 chat template 后调用 `model.generate()`，并只解码输出中超出 prompt 长度的那一段，避免把用户问题重新打印出来。

## 2. 贪心解码

当 `do_sample=False` 时，生成采用最大概率 token：

```math
x_{t+1} = \arg\max_i p_i
```

它的优点是可复现，适合固定题集、回归测试和对比训练版本。缺点是回答可能更保守，局部重复也更容易固定下来。

项目的 `RUN_CONFIG` 可以写为：

```python
do_sample=False
```

此时脚本不会把 `temperature`、`top_p` 传给 `generate()`。

## 3. temperature

采样时先用温度 `\tau` 调整 logits：

```math
p_i^{(\tau)}
= \frac{e^{z_i/\tau}}{\sum_j e^{z_j/\tau}}
```

含义：

- `0 < \tau < 1`：分布更尖锐，偏向高概率 token。
- `\tau = 1`：原始分布。
- `\tau > 1`：分布更平坦，输出更多样但更不稳定。

项目要求采样时 `temperature > 0`。默认 `0.7` 是相对稳定的起点。

## 4. nucleus sampling

把 token 按概率从高到低排序，选择最小集合 `V_p`，使其累计概率至少为 `p`：

```math
\sum_{i \in V_p} p_i \ge p
```

然后在该集合内重新归一化并采样：

```math
\tilde{p}_i =
\begin{cases}
\frac{p_i}{\sum_{j \in V_p}p_j}, & i \in V_p \\
0, & i \notin V_p
\end{cases}
```

`top_p=0.9` 表示只在累计概率覆盖 90% 的候选中随机选择。它会随当前分布自动调整候选数量：模型很确定时集合小，模型不确定时集合较大。

## 5. 如何选配置

| 场景 | `do_sample` | `temperature` | `top_p` |
| --- | --- | --- | --- |
| 固定评测、回归比较 | `False` | 不使用 | 不使用 |
| 普通问答 | `True` | `0.6` 到 `0.8` | `0.85` 到 `0.95` |
| 创意表达 | `True` | `0.8` 到 `1.0` | `0.9` 到 `0.95` |
| 内容跑偏 | `True` | 先降低 | 适度降低 |

先固定一组问题和推理配置，再比较不同 adapter。否则输出差异可能来自随机采样，而不是训练质量。
