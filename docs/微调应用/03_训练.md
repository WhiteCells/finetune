# 训练

训练只需要一条命令：

```bash
uv run train.py
```

脚本固定读取 `config/train.yaml` 和 `config/lora.yaml`。所有训练修改都在这两个 YAML 中完成。

## 1. 训练前先改这几项

`config/train.yaml`：

```yaml
model_name_or_path: "./models/Qwen3-4B-Instruct-2507"
train_file: "data/train.jsonl"
eval_file: "data/eval.jsonl"
output_dir: "outputs/qwen3-4b-lora-exp001"
logging_dir: "logs/qwen3-4b-lora-exp001"
run_name: "qwen3-4b-lora-exp001"
```

每次正式实验换一个 `output_dir`、`logging_dir` 和 `run_name`。这样旧 adapter、checkpoint 和 TensorBoard 曲线不会互相覆盖。

不要把 `model_name_or_path` 设为 `Qwen3-4B-Instruct-2507-FP8`。该 fine-grained FP8 格式在当前 Transformers 5.12.1 中只能推理，Trainer 会拒绝其 LoRA 训练；微调请使用非 FP8 的 `Qwen3-4B-Instruct-2507`。

`config/lora.yaml` 的默认起点：

```yaml
r: 16
alpha: 32
dropout: 0.05
target_modules:
  - "q_proj"
  - "k_proj"
  - "v_proj"
  - "o_proj"
```

先保持 attention 的四个投影层。效果不足且验证集没有恶化时，再考虑加入 `gate_proj`、`up_proj`、`down_proj`。

## 2. 冒烟训练

第一次先临时修改 `config/train.yaml`：

```yaml
train_file: "data/smoke_train.jsonl"
eval_file: "data/smoke_eval.jsonl"
output_dir: "outputs/smoke-qwen3-4b-lora"
logging_dir: "logs/smoke-qwen3-4b-lora"
max_length: 512
gradient_accumulation_steps: 1
num_train_epochs: 1
max_steps: 2
logging_steps: 1
save_steps: 1
eval_steps: 1
save_total_limit: 2
run_name: "smoke-qwen3-4b-lora"
```

运行 `uv run train.py`。冒烟训练只验证路径、显存、数据和保存流程，不用于产出模型。

正常时会看到：

- CUDA、Torch、Transformers 和 PEFT 的环境摘要。
- tokenizer、基座模型和 LoRA 可训练参数摘要。
- 训练集样本数。
- 两步训练后的 `adapter_model.safetensors` 和 `adapter_config.json`。

## 3. 重要参数

| 字段 | 默认值 | 调整建议 |
| --- | --- | --- |
| `max_length` | `2048` | 显存不足时先降到 `1024` 或 `512` |
| `per_device_train_batch_size` | `1` | 单卡显存紧张时保持 1 |
| `gradient_accumulation_steps` | `8` | 增大可提升有效 batch，但训练更慢 |
| `learning_rate` | `2.0e-4` | loss 发散时先降到 `1.0e-4` 或 `5.0e-5` |
| `num_train_epochs` | `3` | 小数据先从 1 开始 |
| `warmup_ratio` | `0.03` | 训练初期不稳可调到 `0.05` |
| `gradient_checkpointing` | `true` | 省显存但会变慢 |

单卡的有效 batch size 是：

```text
per_device_train_batch_size * gradient_accumulation_steps
```

当前默认是 `1 * 8 = 8`。更完整的推导见 [训练优化与显存](../公式细节/03_训练优化与显存.md)。

## 4. 训练产物

以 `output_dir: outputs/qwen3-4b-lora-exp001` 为例：

```text
outputs/qwen3-4b-lora-exp001/
├── adapter_config.json
├── adapter_model.safetensors
├── checkpoint-*/
├── config_snapshots/
├── metrics.summary.json
├── train.log
└── trainer_state.json
```

`adapter_model.safetensors` 不是完整模型。它必须和相同的基座模型一起使用，或通过 `merge_lora.py` 合并。

## 5. 断点续训与旧 adapter

不再通过启动参数覆盖配置。直接修改 `config/train.yaml`：

```yaml
resume_from_checkpoint: "outputs/qwen3-4b-lora-exp001/checkpoint-200"
adapter_path: null
```

这会恢复 optimizer、scheduler 和训练步数，适合中断后继续同一次实验。

只有 adapter、想新开一次训练时：

```yaml
resume_from_checkpoint: null
adapter_path: "outputs/old-adapter"
```

这会加载旧 adapter 的权重，但重新创建 optimizer 和 scheduler。两项不要同时设置。

## 6. 看哪些指标

| 指标 | 正常信号 | 需要注意 |
| --- | --- | --- |
| `loss` | 总体下降，允许短期波动 | 长时间不降、突然成为 `nan` |
| `perplexity` | 通常随 loss 下降 | 极大或持续上升 |
| `eval_loss` | 训练后稳定或下降 | train loss 下降而 eval loss 上升，可能过拟合 |
| `grad_norm` | 有波动 | 连续异常尖峰后 loss 发散 |

TensorBoard 日志在 `logging_dir`。有验证集才会产生有效的 `eval_loss`。
