# 数据准备

目标：让每条样本都能形成“上下文 -> assistant 答案”的监督信号。

## 1. 可直接训练的格式

`trainer/dataset.py` 会在读取时自动识别以下格式，因此小数据集可以直接把文件路径填入 `config/train.yaml` 的 `train_file`。

Alpaca：

```json
{"instruction":"请解释 LoRA。","input":"","output":"LoRA 是一种参数高效微调方法。"}
```

OpenAI messages：

```json
{"messages":[{"role":"user","content":"你好"},{"role":"assistant","content":"你好。"}]}
```

ShareGPT：

```json
{"conversations":[{"from":"human","value":"你好"},{"from":"gpt","value":"你好。"}]}
```

简单自定义单轮：

```json
{"prompt":"你好","response":"你好。"}
```

文件可以是 `.jsonl`，也可以是包含对象或对象列表的 `.json`。

## 2. 统一为 JSONL

数据来自多个来源、需要去重或希望先发现坏样本时，使用预处理脚本。打开 `data/preprocess.py`，修改顶部常量：

```python
INPUT_PATH = Path("data/raw.jsonl")
OUTPUT_PATH = Path("data/train.jsonl")
INPUT_FORMAT = "auto"
SYSTEM_PROMPT = ""
SKIP_INVALID = True
DEDUPLICATE = True
ENSURE_ASCII = False
```

然后运行：

```bash
uv run data/preprocess.py
```

脚本会写出每行一个对象的 JSONL：

```json
{"messages":[{"role":"user","content":"..."},{"role":"assistant","content":"..."}]}
```

并打印输入数、写出数、跳过数、去重数和前两条预览。

## 3. 预处理选项的含义

| 常量 | 作用 |
| --- | --- |
| `INPUT_FORMAT` | `auto` 自动识别；也可固定为 `alpaca`、`sharegpt`、`messages` 或 `custom` |
| `SYSTEM_PROMPT` | 非空时，只给原本没有 system 消息的样本补充 system 消息 |
| `SKIP_INVALID` | 为 `True` 时跳过坏样本并输出原因；为 `False` 时首条坏样本立即报错 |
| `DEDUPLICATE` | 按标准化后的完整 `messages` 内容去重 |
| `ENSURE_ASCII` | 为 `True` 时把中文转义；通常保持 `False` 更便于人工检查 |

## 4. 数据必须满足的约束

项目会拒绝以下样本：

- 消息为空。
- 没有 `assistant` 消息。
- 最后一条消息不是 `assistant`。
- assistant 内容为空。
- 截断后没有任何 assistant token 可计算 loss。

这些限制不是格式洁癖。监督微调要把答案作为标签；只有问题没有答案的样本无法训练。

## 5. 训练集和验证集

正式实验建议准备两个文件：

```text
data/train.jsonl
data/eval.jsonl
```

然后修改：

```yaml
train_file: "data/train.jsonl"
eval_file: "data/eval.jsonl"
```

没有验证集时保持：

```yaml
eval_file: null
```
