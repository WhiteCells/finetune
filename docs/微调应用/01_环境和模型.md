# 环境和模型

目标：准备 Python 环境，并让 `config/train.yaml` 中的模型路径真实存在。

## 1. 安装依赖

项目要求 Python 3.11 或 3.12。进入项目根目录后执行：

```bash
uv sync
```

`uv` 会在项目内创建 `.venv`。之后统一用 `uv run 文件.py` 运行项目脚本，不需要手动激活虚拟环境。

## 2. 下载模型

打开 `download_model.py`，确认顶部几个值：

```python
REPO_ID = "Qwen/Qwen3-4B-Instruct-2507"
OUTPUT_DIR = Path("models") / REPO_ID.split("/")[-1]
REVISION = None
TOKEN = None
```

私有模型可把 `TOKEN` 改为自己的访问令牌。不要把包含令牌的文件提交到版本库。

运行：

```bash
uv run download_model.py
```

默认目录是：

```text
models/Qwen3-4B-Instruct-2507/
```

目录中至少应有 `config.json`、tokenizer 文件和模型权重。下载完成后，`config/train.yaml` 的 `model_name_or_path`、`inference.py` 和 `merge_lora.py` 的 `RUN_CONFIG` 应指向同一个基座模型。

`Qwen3-4B-Instruct-2507-FP8` 是推理量化模型。当前项目使用的 Transformers 5.12.1 不支持对 fine-grained FP8 权重进行 LoRA 训练；它可用于 `main.py` 推理，但 `train.yaml` 必须指向非 FP8 的 `Qwen3-4B-Instruct-2507`。

## 3. 检查 GPU

训练 4B 模型通常需要 CUDA GPU。以下检查不会加载模型：

```bash
uv run python -c "import torch; print(torch.cuda.is_available()); print(torch.cuda.get_device_name(0) if torch.cuda.is_available() else 'CPU')"
```

支持 bf16 的 GPU 使用默认配置：

```yaml
bf16: true
fp16: false
torch_dtype: "bfloat16"
```

较旧的 GPU 不支持 bf16 时，改为：

```yaml
bf16: false
fp16: true
torch_dtype: "float16"
```

CPU 适合预处理和测试；4B 模型的训练或合并通常不适合在 CPU 上完成。

## 4. 最小检查

配置能读取，说明 YAML 字段名没有拼错：

```bash
uv run python -c "from trainer.trainer import load_train_config; from model.lora import load_lora_config; print(load_train_config('config/train.yaml')); print(load_lora_config('config/lora.yaml'))"
```
