# 测试和检查

## 1. 运行测试

```bash
uv run python -m unittest discover -s tests -v
```

当前测试不加载 4B 权重，覆盖：

- 数据格式识别、标准化、去重和跳过坏样本。
- LoRA adapter 推理配置与采样参数。
- tokenizer 词表变化时的 embedding 处理。
- 训练 YAML 的关键参数校验。

## 2. 语法检查

```bash
uv run python -m compileall -q data model trainer utils tests train.py inference.py merge_lora.py download_model.py main.py
```

无输出表示语法检查通过。

## 3. 不加载模型的配置检查

```bash
uv run python -c "from trainer.trainer import load_train_config; from model.lora import load_lora_config; print(load_train_config('config/train.yaml')); print(load_lora_config('config/lora.yaml'))"
```

它能发现 YAML 缩进、未知字段、LoRA rank 或精度设置的基础问题。

## 4. 真实闭环检查

按以下顺序缩小风险：

1. 使用 `data/example.jsonl`。
2. 临时设定 `max_length: 512` 和 `max_steps: 2`。
3. 运行 `uv run train.py`。
4. 在 `inference.py` 填入刚生成的 adapter 路径。
5. 运行 `uv run inference.py`。

这个闭环能确认数据、模型、LoRA、保存和推理格式是一致的。确认通过后，再换成真实训练集、验证集和正式输出目录。
